# wechat-voiceover-skill

一个 AI Agent Skill，将公众号文章（或任何书面长文）改写成适合录音的口播稿。

## 它是什么

这是一个 AI 写作技能文件（SKILL.md），专门用于：

- 把书面文章转成"为耳朵而写"的口播稿
- 产出可直接喂给语音合成工具（TTS）的纯文本
- 适用于视频号、B站视频配音、播客录制等场景

## 解决什么问题

书面稿直接念会"水土不服"：

| 问题 | 症状 |
|---|---|
| 标题符号被读出来 | `## 三、风险` 被合成器念成乱码 |
| 长句跟不上 | 三层修饰套一句，耳朵听不清 |
| 换人听懵 | 书面的"他/她"在口播中无法区分 |
| 缩略词读错 | "化生放核"合成器不知道怎么念 |
| 开头没钩子 | 书面引言太慢，听众直接划走 |

本 Skill 系统性地处理这些问题，产出一份干净、可直接录音的口播稿。

## 工作流定位

本技能通常接在 [wechat-breakdown-skill](https://github.com/iHamburg/wechat-breakdown-skill) 之后：

```
书籍/视频 → [wechat-breakdown-skill] → 公众号文章
                                            ↓
                              [wechat-voiceover-skill] → 口播稿
                                                            ↓
                                                    TTS语音合成 → 音频
                                                            ↓
                                                    + 风景视频 → 发布
```

## 三类核心改动

### A. 结构性删改
- 删除标题、分隔线、加粗等所有 markdown 符号
- 小标题改写为口语过渡句
- 括号补充信息并入正文或删除

### B. 书面词转口语
- 换人处加明确引导（"接下来这位是某某"）
- 长句拆短，加缓冲词（"你想""也就是说"）
- 并列项显式编号
- 缩略词展开、数字标注读法

### C. 为"听"而加
- 开场钩子（前几秒抓住人）
- 段间口语过渡
- 自然收束结尾

## 如何使用

### Cursor / Claude Code

```bash
# Cursor
cp SKILL.md ~/.cursor/skills/wechat-voiceover/SKILL.md

# Claude Code
cp SKILL.md ~/.claude/skills/wechat-voiceover/SKILL.md
```

然后在对话中说：

> "把这篇文章转成口播稿"

> "改成适合念的版本"

> "做成喂给 TTS 的稿子"

### 手动使用

将 `SKILL.md` 内容作为 system prompt 喂给任何 LLM，再提供你的文章即可。

## 触发条件

- "把这篇转成口播稿"
- "改成口播 / 配音稿 / 旁白"
- "转成适合念的版本"
- "做成喂给 TTS / 语音合成的稿子"
- 在已有书面文章、准备做成音频或视频时

## 输出规格

- 纯文本 `.txt`（零 markdown 符号）
- 附带：估算时长、英文词列表、需确认的数字读法
- 中文口播约 250-300 字/分钟

## 对照示例

**书面原文（不能直接念）：**

> \#\# 三、但风险，是这条路上的地雷
> \*\*他真正最忧虑的，是权力的集中与滥用。\*\*

**口播稿（这样改）：**

> 把好的一面讲完，他话锋一转，开始讲风险。而且他最担心的，并不是科幻片里那种失控。他真正最忧虑的，是权力的集中和滥用。
