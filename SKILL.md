# 公众号文章转口播稿

把一篇书面的公众号文章,改写成一份"为耳朵而写"的口播稿。

核心认知:**书面是给眼睛读的,口播是给耳朵听的**。读者能回看、能跳读、能看见排版;听众只能线性地、一次性地往下听。本技能所有改动,都服务于这个差别。

本技能通常接在 wechat-book-breakdown 之后:产出的口播稿,再进入"语音合成 → 自动转字幕 → 拼风景视频"的流程。所以它产出的稿子,必须对语音合成器足够"干净"。

## 一、动手前先确认两件事(避免返工)

除非用户已经说明,否则**先确认再写**——这两点直接决定怎么改:

1. **时长 / 取舍**:全量口语化(保留全部内容),还是精简到某个时长?给个换算帮用户判断——中文口播大约**每分钟 250–300 字**,所以一篇约 4000 字的稿子,念出来大约 14–16 分钟。视频号偏短(3–8 分钟),B站可长。
2. **结尾要不要加引导语**(如"点赞关注""评论区聊聊")。**不要默认加**,不少博主不要。

视角默认用**博主转述体**("他认为……""她举了个例子"),比书面的纯第三人称更适合口播;用户另有要求再调整。

## 二、三类改动(本技能的核心)

### A. 结构性删改(整段处理)
口播里不存在"版面",以下一律清掉或改写:
- **主标题、备选标题块**:删。开头另起一句口语引入。
- **所有小标题(## 一、二……)**:不能直接念出来。改写成口语过渡句。例:"## 三、风险" → "把好的一面讲完,他话锋一转,开始讲风险"。
- **分隔线 `---`、`**加粗**`、以及 `「」""()【】` 等所有符号**:全部去掉。括号里的补充信息,要么并进正文顺成一句,要么删。
- **"结语""引言"这类版面词**:删,直接进入收尾。
- **文末出处 / 版权说明**:删(口播不读)。

### B. 书面词 → 口语(听感)
- **换说话人时补明确引导**:书面靠"他/她"区分人没问题,但听觉上极易混。每次切换人物,加一句"我们先说第一位,某某""接下来这位,是某某""最后登场的,是某某"。
- **长定语 / 从句拆成短句**:一句里套三层修饰,耳朵跟不上。拆成两三个短句,必要时用"你想""也就是说""怎么理解呢"来缓冲。
- **并列项显式编号**:一连串并列特征,改成"第一……第二……第三……最关键的是最后一点……",听众才数得清。
- **缩略词、生造词展开**:例如"化生放核" → "化学、生物、放射、核武器这些领域"。
- **去掉纯书面连接词**:如下、上述、如前所述、综上、详见——口播都不用。
- **数字标注读法**:把易读错的数字写成口播读法,如"452 期" → "第四百五十二期","2026 年" → "二零二六年",避免合成器读错。

### C. 为"听"而加(书面没有、口播需要)
- **开场钩子**:前几秒抓不住人就会被划走。开头加一句问题或悬念,把全篇最大的张力先抛出来。
- **段间口语过渡**:用"那么""说到这里""我们回过头来看""讲完了某某,我们说下一位"承上启下,替代书面里的分隔线和小标题。
- **结尾收束**:自然收尾即可;是否加引导语,按第一节确认的结果。

## 三、面向语音合成(TTS)的特别要求

产出会直接喂给语音合成工具,所以:
- **纯文本内容,存为 `.md`**:文件扩展名用 `.md`(方便在 Obsidian 等笔记工具中管理),但**文件内容本身不得使用任何 markdown 格式符号**(`#`、`*`、`---`、`>`、`「」` 等)。段落之间用一个空行分隔即可。
- **英文 / 品牌词的两难**:合成器对英文,尤其是夹在中文里的品牌名(如 Claude、Anthropic),读法常常不准。两种处理任选其一,**写完要主动提示用户**:
  - 保留原文(适合读者熟悉的品牌),并提醒用户合成后重点听这几处;
  - 或额外出一个"全中文读音版",把英文换成中文音译或口头叫法(Claude→克劳德,某公司→"这家公司"),专门喂 TTS。
- **写完做自检,并把结果报告给用户**:扫一遍全文,告诉用户——中文字数、估算时长、是否还残留 `#`/`*`/`---`/`「」` 等符号、以及出现了哪些英文词。

## 四、产出与交付

- 产出 **`.md` 文件**(内容为纯文本,零格式符号),放到输出目录,用 present_files 交付,方便用户在笔记工具中管理,也可直接粘进语音合成工具。
- 交付时一并给出:估算时长、需要用户人工确认的点(数字读法、英文词读法),并提醒一句:**先通听一遍再去转字幕**,合成器偶有吞字或怪读,先听一遍能省去返工。

## 五、好 / 坏 对照

**书面原文(不能直接念):**

> \#\# 三、但风险,是这条路上的地雷
> \*\*他真正最忧虑的,是权力的集中与滥用。\*\* 人工智能会让世界的"权力总量"变大……在化生放核领域被滥用。

**口播稿(这样改):**

> 把好的一面讲完,他话锋一转,开始讲风险。而且他最担心的,并不是科幻片里那种失控。他真正最忧虑的,是权力的集中和滥用。他说,人工智能会让这个世界的"权力总量"变大……(后文把"化生放核"展开成"化学、生物、放射、核武器这些领域"。)

差别:删掉了标题和所有符号、换人处加了过渡和"他说"、把缩略词展开、句子拆得更短更顺口。这就是本技能要的效果。
